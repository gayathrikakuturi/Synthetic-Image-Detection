Lightweight design makes it ideal for real-time image classification without sacrificing deep feature learning.

How GradCAM Works
🔵
Cold Zones (Blue)
Low attention regions. Minimal contribution to the model's prediction — typically background areas.
🟡
Warm Zones (Yellow/Green)
Moderate activation. Edges and textures that partially influence the decision.
🔴
Hot Zones (Red)
Critical regions driving the prediction — synthetic artifacts, unnatural patterns, key discriminating features.

Feature Extraction Flow
Step-by-Step
Processing Pipeline Walkthrough
01
📤
Image Received via Flask Route
The /analyze endpoint receives either an uploaded file or a base64 webcam capture. Bytes are decoded using NumPy and OpenCV into a BGR numpy array.
cv2.imdecode(np.frombuffer(file.read(), np.uint8), cv2.IMREAD_COLOR)
02
⚙️
Preprocessing
Image is converted BGR→RGB, resized to 224×224, normalized to [0,1], and given a batch dimension.
img = cv2.resize(img,(224,224)) / 255.0 → shape: (1,224,224,3)
03
🧠
MobileNetV2 Inference
The preprocessed image is fed into the model. The sigmoid output is a probability between 0 and 1. Above 0.5 = Real, below = Fake.
prob = model.predict(processed)[0][0] → "Real" if prob > 0.5
04
🗺️
GradCAM Heatmap Generation
GradientTape records gradients of the loss w.r.t. the last Conv2D layer. Pooled gradients are multiplied with feature maps and summed to create a heatmap, then colorized using COLORMAP_JET.
heatmap = ReLU(Σ αk × Ak) → resize → applyColorMap → addWeighted
05
🔬
Feature Extraction & Explanation
Four numerical features are computed from the image and heatmap. Threshold-based rules generate the human-readable explanation.
texture > 60 → "Irregular texture" | edge > 60 → "Edge artifacts"
06
📊
Base64 Encoding & Rendering
Both the original image and GradCAM overlay are JPEG-encoded and Base64-converted, then passed as Jinja2 variables to result.html for inline display.
base64.b64encode(buffer).decode() → render_template("result.html", data=result)
Detection Signals
Real vs. Synthetic: Key Differences
🌀
Texture Patterns
Real photos have organic natural texture variation. AI images often have overly smooth or repetitively patterned textures from upsampling artifacts.
📐
Edge Consistency
Real images have edges following natural physics. Synthetic images may show boundary artifacts, blurred edges, or inconsistent sharpness.
💡
Lighting & Shadows
Real images have consistent lighting directions. GAN/diffusion-generated images often contain subtle lighting inconsistencies across regions.
🌄
Background Coherence
AI-generated content may have model attention concentrating unnaturally on background regions, revealing synthesis artifacts.
👁️
Fine Details
Teeth, hair strands, fingers, and reflections are areas where generative models frequently fail, producing detectable imperfections.
🔁
Frequency Artifacts
GAN and diffusion models introduce characteristic high-frequency patterns that CNNs like MobileNetV2 learn to detect during training.
